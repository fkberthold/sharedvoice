# How-to guides

How-to guides are the *application + action* quadrant: focused
recipes for the already-competent user who knows what they want to
do. Each guide solves one problem; titles state the task precisely;
steps are executable.

If you are learning the project for the first time, start at the
[Tutorials](../tutorials/index.md) instead. For surface catalogues,
see the [Reference](../reference/index.md). For the *why* behind the
design, see the [Explanation](../explanation/index.md).
